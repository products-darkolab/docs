## Workflow

All work is tracked in [Jira](https://darkolab.atlassian.net/jira/software/projects/DLAB). Every ticket maps to one commit.

```
DLAB-XX ticket created
  → Claude Code session → implementation
  → commit: "feat(DLAB-XX): Short description [DLAB-XX]"
  → Railway auto-deploys API
  → GitHub Actions deploys frontend to Vercel
```

## Commit Convention

```
feat(DLAB-26): Cohort Matching Algorithm [DLAB-26]
fix(DLAB-22): Correct asyncpg multi-statement bug in migration
chore: update requirements.txt
docs: add DLAB-26 to changelog
```

## Key Technical Rules

These are the most common sources of bugs. Read before writing any code:

<AccordionGroup>
  <Accordion title="asyncpg multi-statement rule">
    Never use `;` to chain SQL in a `text()` call. asyncpg will throw `cannot use multiple statements in query`. Always use the `STATEMENTS = [...]` list pattern in migrations.
  </Accordion>
  <Accordion title="bcrypt version">
    Always use `bcrypt==4.0.1`. It is pinned in requirements.txt. Later versions broke the API used by passlib.
  </Accordion>
  <Accordion title="Enrollment access_type">
    `Enrollment` has a non-nullable `access_type` column. Always pass `access_type="free"` when creating an Enrollment record, unless the user has a Pro subscription.
  </Accordion>
  <Accordion title="User model field names">
    The User model uses `full_name` (not `first_name` + `last_name`). XpHistory uses `xp_amount` and `reason` (not `amount` and `source`).
  </Accordion>
  <Accordion title="No internal HTTP calls">
    Never call your own API via HTTP from within the API. Call service functions directly. Example: `await assign_peer_review(db, ...)` not `await httpx.post('/v1/cohorts/peer-reviews/assign')`.
  </Accordion>
  <Accordion title="Frontend API client">
    Always use `apiClient` (axios with interceptors) not raw `fetch`. The interceptor handles auth tokens automatically. Import from `@/lib/api/client`.
  </Accordion>
  <Accordion title="No Anthropic SDK">
    Never use the Anthropic Python SDK. Use raw httpx via `BaseAgent.call_claude()`. This gives us full control over retries, timeouts, and cost tracking.
  </Accordion>
</AccordionGroup>

## File Structure

```
safua-platform/
├── api/
│   ├── app/
│   │   ├── agents/       # BaseAgent + all AI agents
│   │   ├── models/       # SQLAlchemy models
│   │   ├── routers/      # FastAPI route handlers
│   │   ├── services/     # Business logic (industry pulse, cohort matching)
│   │   ├── scripts/      # Seed scripts, admin utilities
│   │   ├── config.py     # Settings via pydantic
│   │   ├── database.py   # DB engine + session factory
│   │   ├── dependencies.py # get_current_user, require_admin
│   │   └── main.py       # FastAPI app + scheduler + router registration
│   ├── migrations/       # One file per DB migration
│   └── run_migrations.py # Migration runner
└── platform/
    ├── src/
    │   ├── app/          # Next.js app router pages
    │   ├── components/   # Shared React components
    │   └── lib/
    │       └── api/      # apiClient + per-router API modules
    └── public/
```
