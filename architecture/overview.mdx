## Key Design Decisions

### Raw httpx for Claude API
All agent calls use raw `httpx` (not the Anthropic SDK). This gives us full control over retry logic, streaming, and cost tracking. Every agent extends `BaseAgent` in `api/app/agents/base_agent.py`.

```python
# Pattern: never do this
import anthropic
client = anthropic.Anthropic()

# Pattern: always do this
response = await httpx_client.post(
    "https://api.anthropic.com/v1/messages",
    headers={"x-api-key": ANTHROPIC_API_KEY, "anthropic-version": "2023-06-01"},
    json={...}
)
```

### Async SQLAlchemy
All database operations are async using `asyncpg` driver. Key rule: **never use multiple SQL statements in a single `text()` call** — asyncpg doesn't support it. Always split on `;` and execute each statement separately.

### Two-Tier Migration System
We use a custom `run_migrations.py` on top of Alembic. All new features add a migration file to `api/migrations/` and register it in `MIGRATION_ORDER`. Every migration must be **idempotent** (use `IF NOT EXISTS`).

### No Internal HTTP Calls
Services call each other's functions directly — never via HTTP. For example, when a lab is submitted, `assign_peer_review()` is called directly from the code review router, not via `POST /v1/cohorts/peer-reviews`.

## Services Overview

| Service | Location | Purpose |
|---|---|---|
| Industry Pulse | `api/app/services/industry_pulse.py` | Scrapes job boards, GitHub, arXiv; scores skill freshness |
| Cohort Matching | `api/app/services/cohort_matching.py` | Groups learners, assigns peer reviews, computes health |
| Agent Orchestrator | `api/app/agents/orchestrator.py` | Routes requests to sub-agents, injects user memory |

## Scheduled Jobs (APScheduler)

| Job | Schedule | What it does |
|---|---|---|
| Industry Pulse | Daily 3am UTC | Fetches signals, recomputes freshness scores |
| Cohort Matching | Sunday midnight UTC | Matches unmatched learners into cohorts |
| Challenge Rotation | Monday 1am UTC | Deactivates expired challenges, seeds new ones |
