## Architecture

Safua uses a multi-agent system built on top of Claude Sonnet 4. All agents share a `BaseAgent` class and communicate through a `MetaOrchestrator`.

```
User Request
     │
     ▼
MetaOrchestrator (api/app/agents/orchestrator.py)
     │  - Injects top 10 user memories
     │  - Routes to correct sub-agent
     │
     ├──▶ SocraticTutorAgent    — lesson help, concept questions
     ├──▶ CodeReviewAgent       — lab submission review
     └──▶ MotivationAgent       — engagement nudges, streak alerts
```

## BaseAgent

All agents extend `BaseAgent` in `api/app/agents/base_agent.py`.

```python
class BaseAgent:
    MODEL = "claude-sonnet-4-20250514"
    MAX_TOKENS = 1000

    async def call_claude(self, system: str, messages: list[dict]) -> str:
        """Raw httpx call to Anthropic API. Never use the SDK."""
        response = await httpx_client.post(
            "https://api.anthropic.com/v1/messages",
            headers={
                "x-api-key": ANTHROPIC_API_KEY,
                "anthropic-version": "2023-06-01",
                "content-type": "application/json",
            },
            json={
                "model": self.MODEL,
                "max_tokens": self.MAX_TOKENS,
                "system": system,
                "messages": messages,
            },
            timeout=30.0,
        )
        data = response.json()
        return data["content"][0]["text"]
```

<Warning>
Never use the Anthropic Python SDK. Always use raw httpx. This ensures full control over retry logic, streaming, timeout, and cost tracking.
</Warning>

## SocraticTutorAgent

The Socratic Tutor never gives answers directly. It asks guiding questions until the learner arrives at the answer themselves.

**Behaviour rules (baked into system prompt):**
- Never reveal the answer
- Give a hint after 3 incorrect attempts
- Reference the learner's skill mastery level to calibrate difficulty
- Praise effort, not intelligence

**Socratic Mode** (toggle on lesson page):
- Off: Direct answers with explanation
- On: Pure Socratic — only guiding questions

**API:** `POST /v1/lesson-tutor/chat`

## CodeReviewAgent

Triggered automatically when a learner submits a lab. Returns structured JSON:

```json
{
  "overall_score": 82,
  "correctness": {"score": 90, "feedback": "..."},
  "style": {"score": 75, "feedback": "..."},
  "efficiency": {"score": 80, "feedback": "..."},
  "inline_comments": [
    {"line": 14, "type": "hint", "comment": "..."},
    {"line": 22, "type": "question", "comment": "..."},
    {"line": 31, "type": "praise", "comment": "..."}
  ],
  "socratic_questions": ["...", "..."]
}
```

Inline comments are rendered in the Monaco editor using glyph margin decorations:
- 🔴 Red glyph — hint
- 🔵 Blue glyph — Socratic question
- 🟢 Green glyph — praise

**API:** `POST /v1/code-review/submit`

## MotivationAgent

Sends in-app notifications triggered by 4 conditions:

| Trigger | Condition | Message Style |
|---|---|---|
| `absence_48hr` | No activity in 48 hours | Gentle check-in |
| `streak_at_risk` | Streak expires in < 4 hours | Urgency nudge |
| `league_push` | Within 10 XP of rank-up | Competitive encouragement |
| `skill_decay` | Skill mastery dropped below 0.5 | Learning reminder |

**API:** `GET /v1/notifications` (polled every 5 minutes by TopBar)

## User Memory

The orchestrator injects the top 10 relevant memories into every agent prompt. Memories are stored in `user_agent_memory`:

```python
# Stored by agents after interactions
{
  "user_id": "...",
  "memory_type": "struggle",        # strength | struggle | preference | context
  "content": "Struggles with async/await mental model",
  "skill_slug": "python-async",
  "confidence": 0.85
}
```

This gives every agent persistent, personalised context across sessions without bloating the context window.
