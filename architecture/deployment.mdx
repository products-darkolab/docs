## Overview

Safua runs on two cloud services connected by a GitHub monorepo:

```
github.com/products-darkolab/safua-platform
├── platform/    ──▶  Vercel (safua.ai)
└── api/         ──▶  Railway (api.safua.ai)
```

## Frontend — Vercel

The Next.js frontend deploys to Vercel on every push to `main`.

**Why GitHub Actions instead of Vercel's native deploy?**
Vercel's native GitHub webhook triggers a build for every file in the repo, not just `platform/`. We use a custom GitHub Actions workflow that only triggers when files in `platform/` change, and uses the Vercel CLI directly.

```yaml
# .github/workflows/vercel-deploy.yml
on:
  push:
    branches: [main]
    paths: ['platform/**']

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Deploy to Vercel
        run: npx vercel --prod --token ${{ secrets.VERCEL_TOKEN }}
        working-directory: platform
```

<Warning>
Vercel's "Ignored Build Step" in project settings must be configured to only build commits from the correct author. Otherwise Vercel will also try to build via its own webhook, causing double deploys.
</Warning>

**Environment variables (set in Vercel dashboard):**
```
NEXT_PUBLIC_API_URL=https://api.safua.ai/v1
NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY=pk_live_...
```

## Backend — Railway

The FastAPI backend runs on Railway with automatic deploys from the `main` branch.

Railway is configured to:
1. Build from the `api/` subdirectory
2. Run `python run_migrations.py && uvicorn app.main:app --host 0.0.0.0 --port $PORT` as the start command
3. Provision a managed PostgreSQL database

**Environment variables (set in Railway dashboard):**
```
DATABASE_URL=postgresql+asyncpg://...
SECRET_KEY=...
ANTHROPIC_API_KEY=sk-ant-...
STRIPE_SECRET_KEY=sk_live_...
STRIPE_WEBHOOK_SECRET=whsec_...
RESEND_API_KEY=re_...
FRONTEND_URL=https://safua.ai
```

<Note>
`DATABASE_URL` must use the `postgresql+asyncpg://` scheme (not `postgresql://`) for SQLAlchemy async to work.
</Note>

## Database — Railway PostgreSQL

PostgreSQL is provisioned by Railway and co-located with the API service. Connection is via the internal Railway network (low latency, no egress charges).

Migrations run automatically on every deploy via `run_migrations.py`. All migrations are idempotent — safe to run multiple times.

## Domain Setup

| Domain | Service | DNS |
|---|---|---|
| safua.ai | Vercel | CNAME → cname.vercel-dns.com |
| www.safua.ai | Vercel | CNAME → cname.vercel-dns.com |
| api.safua.ai | Railway | CNAME → Railway-provided hostname |
| docs.safua.ai | Mintlify | CNAME → hosting.mintlify.com |

## Secrets Management

All secrets are managed per-service:
- **Vercel**: Dashboard → Project → Settings → Environment Variables
- **Railway**: Dashboard → Service → Variables
- **GitHub Actions**: Repository → Settings → Secrets and variables → Actions

Never commit `.env` files. Use `.env.example` with placeholder values only.
