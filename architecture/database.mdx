## Overview

Safua uses a single PostgreSQL database (hosted on Railway) with 30+ tables. All IDs are UUIDs generated with `gen_random_uuid()`. All timestamps are `TIMESTAMP WITH TIME ZONE`.

## Learning Content

```
roadmaps → levels → modules → lessons → labs
```

- **levels**: order within a roadmap
- **modules**: grouped lessons, has `skill_slugs` JSONB
- **lessons**: type `reading` or `lab`, has `content` TEXT
- **labs**: `instructions`, `starter_code`, `solution_code`, `skill_slugs` JSONB
- **lab_submissions**: `code`, `ai_review` JSONB (Code Review Agent output)

## Gamification Tables

| Table | Purpose |
|---|---|
| xp_history | Every XP award — `user_id`, `xp_amount`, `reason` |
| streak_wagers | Active wagers — `user_id`, `bet_amount`, `target_streak` |
| focus_freezes | Available freezes per user |
| leagues | Weekly league instances |
| league_memberships | User → league with XP and rank |
| skill_reviews | Spaced repetition review queue (SM-2) |

## Agent Tables

| Table | Purpose |
|---|---|
| user_agent_memory | Per-user facts/preferences/struggles stored by agents |
| notifications | In-app notifications from motivation agent |

## Industry Pulse Tables

| Table | Purpose |
|---|---|
| industry_signals | Raw scraped signals (job postings, GitHub repos, arXiv papers, HN stories) |
| skill_freshness_scores | Computed freshness score per skill_slug (0–100) |

## Cohort Tables

| Table | Purpose |
|---|---|
| cohorts | Cohort groups — `track`, `health_score`, `is_active` |
| cohort_members | User → cohort with `match_score`, `skill_vector`, review/challenge counts |
| cohort_posts | Standup feed posts |
| cohort_challenges | Weekly + monthly challenges per cohort |
| challenge_submissions | Learner submissions for challenges |
| peer_review_assignments | Structured peer code review assignments |
| cohort_health_scores | Weekly health score snapshots |
